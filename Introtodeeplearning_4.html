LiDAR¶

Actualmente los vehculos autnomos se dice que son de Nivel 2, es decir, requieren atencin por parte del conductor para evitar accidentes. Lo que se espera es que en un futuro se llegue a niveles 3 o 4, donde el vehculo ya sera completamente autnomo, requiriendo muy poca atencin por parte del conductor.

LIDAR¶

LiDAR es un sistema de deteccin basado en pulsos de lser. En vehcuos autnomos se tienen dos diferentes escenarios, la conduccin por carretera y conduccin en ciudades, donde cada uno de los diferentes escenarios presentan retos distintos, siendo la conduccin urbana la ms complicada. Para el diseo de un sistema de conduccin autnoma se deben tomar en cuenta diferentes aspectos, por ejemplo, la velocidad, ya que si queremos que el vehculo vaya ms rpido, entonces necesitaremos aumentar el frame rate para obtener imgenes con ms frecuencia y as tomar las mejores decisiones.

Tambin influyen otros aspectos como el diseo del vehculo.

Un sistema LiDAR suele utilizarse al mismo tiempo que una cmara, ya que diferentes factores afectan a la imagen de una cmara y el sistema LiDAR provee mayor informacin.

Para que el vehculo tenga un mejor comportamiento, el sistema debe ser capaz de clasificar imgenes correctamente. Para ello se requiere una buena resolucin de 0.05 para la adquisicin de la imagen.

En estos sistemas, la distancia mnima de deteccin de objetos es la distancia de frenado, en caso de que haya algn objeto en el camino que pueda daar al vehculo. Adems de la distancia de deteccin, es importante la resolucin y el framerate, ya que si no se detecta adecuadamente el objeto o no se detecta a tiempo, entonces no importa qu tan lejos lo pueda detectar el sistema. La resolucin contribuye ms que el framerate.

Detección para evitar colisiones¶

Para evitar colisiones se utiliza un algoritmo donde se toman pares de puntos y si son cercanos y la altura de estos es mayor a un valor, entonces se evitar ir por esos puntos. Los puntos menores a esta altura son aquellos por donde el vehculo puede pasar.

Utilizando LiDAR se detectan objetos muy lejanos con poca informacin, por lo que es posible evitar obstculos que posiblemente no sean 100% reconocibles. Utilizando este sistema se puede obtener informacin sobre el objeto, por ejemplo, si se est moviendo o si es un objeto estacionario.

Una de las cosas que se hace utilizando la informacin proporcionada por LiDAR es la voxelizacin, que es una representacin en elementos tridimensionales. Lo que se quiere es tener la mejor resolucin al menor costo computacional. Para esto, se realiza una fusin de un algoritmo de clustering y una red neuronal. La red neuronal falla en precisin, mientras que el algoritmo clsico suele dar falsos positivos, por lo que la combinacin de ambos resulta en algo ms preciso.

Reconocimiento de voz¶

Para evaluar un sistema de reconocimiento de voz se utiliza como mtrica la transcripcin de una persona. A partir de ello se compara la transcripcin realizada de la red y se obtiene el error. En Rev se compar el desempeo con diferentes sistemas de reconocimiento de voz y se muestra que algunas de las palabras ms complicadas de poder reconocer correctamente son nombres de personas y de compaas

Bias¶

Debido a los diferentes acentos que hay, es que el desempeo del sistema va a cambiar, provocando mayores errores para personas con un cierto acento que otras personas con acentos comunes dentro del conjunto de datos.

Estos sistemas dependen de diferentes variables, el objetivo es lograr elabroar un sistema lo ms amplio posible, de manera que pueda realizar transcripciones independientemente del tipo de contenido del audio, caractersticas de quien est hablando, ruido y condiciones del canal.

Selección de datos y segmentación¶

  • Verbatim: Se requieren transcriptores que realicen correcciones a las transcripciones realizadas por el sistema para obtener un resultado lo ms coherente posible.
  • Los audios se dividen en diferentes oraciones para el entrenamiento, pero al momento de probar el modelo, no se puede dividir el audio en segmentos en tiempo real.

Características de la señal¶

Lo mejor para poder obtener caractersticas de la seal es primero realizar un procesamiento de la misma. Se toma para cada tiempo un vector con la energa de la seal en diferentes frecuencias. Para esta parte se utiliza la escala de Mel para tratar de imitar la forma en la que los humanos procesan el audio.

Encoder - Decoder con atención¶

Es un modelo que puede aplicarse a una variedad de problemas de secuencias de datos.

Capas¶

  • Embedding: Se trata a la seal de audio como una imagen y se utiliza una red convolucional
  • Se utilizan diferentes modelos de redes neuronales para continuar con el procesamiento y clasificacin de la seal.
    • RNNS: se utilizaban comnmente LSTM pero han sido sustitudas por Transformers.
    • Transformers: Mejoran el entrenamiento hacindolo ms eficiente.
    • Conformers: Son transformers con una capa convolucional extra despus de la capa de atencin.

Los modelos basados en atencin no suelen ser muy eficientes en el mbito comercial.

CTCs¶

En lugar de redes basadas en atencin se utiliza Connectionist Temporal Classification. Lo que requiere este algoritmo para funcionar es que la entrada y salida sean monotnicas, que la salida sea igual o de menor longitud que la entrada. Un modelo basado en CTC no tiene realimentacin, se tiene un nmero fijo de salidas.

Los modelos que utilizan CTC no son tan buenos como los modelos con encoder-decoder pero s son mejores que los modelos hbridos (transformers y conformers), adems de que son mejores en el tiempo de decodificacin.

Para mejorar el desempeo de un modelo CTC se puede aadir un modelo de lenguaje, que pueden ser modelos estadsticos o utilizando redes neuronales. Cuando se utilizan estos dos modelos para poder optimizar todos los parmetros, entonces se le llama RNN-T, Transducer o Transducer Transformer.

Modelo CTC/Atención¶

Se puede tomar el modelo de encoder-decoder y agregar una capa de CTC para tener un buen desempeo y se mejora la velocidad del sistema, hacindolo viable para fines comerciales.

Primero entran los datos al encoder y se obtienen salidas mediante CTC, obteniendo una lista de hiptesis; luego se utiliza el decoder, donde entran las hiptesis del CTC y se obtiene la mejor.
